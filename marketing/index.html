Architecture

A small, deliberate stack.

Tauri for the shell, Milkdown on ProseMirror for the editor, Pandoc as a sidecar for format conversion, Lowlight for syntax highlighting via decorations (never DOM mutation).

Shell

Tauri 2 · Rust

Native webview, custom frameless window, single-instance routing, file associations on install.

Editor core

Milkdown · ProseMirror

Live WYSIWYG Markdown. Custom plugins for markdown-aware paste and view-hooks (typewriter + focus).

Format converter

Pandoc · sidecar binary

Bundled ~220 MB on disk, invoked as a separate process. GPL stays at the Pandoc boundary.

Syntax highlighting

Lowlight · Decorations

Tokenize via hljs AST, paint ProseMirror decorations. No DOM mutation — cursor and selection stay honest during edits.

Ctrl+K opens every action in the app. Fuzzy-scored with structural matching. Every shortcut is in the menu; every menu is in the palette.

Design tokens

Two themes, one semantic contract.

Every token — surface, fg, accent, hl — exists in both Obsidian Ink and Ivory Paper. Flip the theme toggle top-right and watch everything rebind.

Obsidian Ink

Deep, warm charcoal.

Off-white body text reads easier than pure white on tired eyes. Indigo for the accent is the same hue you see in Ivory Paper — just brighter.

Ivory Paper

Cream, not screen-white.

A deeper indigo for legibility. Warm near-black text that reads like a freshly printed page.

FAQ

Short answers.

Why Tauri instead of Electron?

Tauri uses the platform's own webview (WebView2 on Windows) instead of bundling Chromium. That's why the installer is 44 MB instead of 120+, and cold launch is ~200 ms instead of a second. The Rust backend is small and handles file dialogs, Pandoc invocation, and single-instance routing.

Why bundle Pandoc? It's huge.

Pandoc is the only realistic answer to "handle 40+ document formats." We invoke it as a sidecar binary (separate process), so the GPL applies to Pandoc itself but doesn't propagate to TypeX. The binary compresses from ~220 MB to ~40 MB inside the MSI. Without Pandoc, users would have to install it separately; the bundled version makes the 40-format promise actually hold on first launch.

How does syntax highlighting work if it doesn't mutate the DOM?

Lowlight (the AST-only variant of highlight.js) tokenizes code into a HAST tree. We walk that tree and emit ProseMirror Decoration.inline objects with hljs-* classes at the right positions. ProseMirror paints the decorations over the text without changing it. The cursor, selection, and edit history stay correct during editing.

Is there telemetry?

None. No analytics, no crash reporting, no usage metrics. Recent files, workspace, prefs, and session are stored in localStorage on your machine and never leave it.

macOS / Linux?

Planned. Windows first because the file-association story is the most mature there; macOS needs a native path for Reminders-style features and Linux has its own desktop integration quirks. Priority in the roadmap.

Can I ship my own themes?

Every visual decision is a CSS custom property prefixed --tx-. Drop a file in <workspace>/.typex/themes/*.css that overrides the tokens and TypeX picks it up on restart. See the design system folder in the repo for the full token list.
