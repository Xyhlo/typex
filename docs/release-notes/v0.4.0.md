**TypeX 0.4.0 — AI for writing, without the chatbot**

Phase 4 of the roadmap ships as one release: an opt-in AI surface that meets you at whatever you already have installed. Local Ollama, direct APIs for Anthropic / OpenAI / Gemini, or chat-agent CLIs (Claude Code / Codex / Gemini CLI) you're already logged into. Seven adapters, one interface, no telemetry.

AI is off by default. You turn it on, you pick a provider, and only then does anything reach out. Prompts stay on your machine unless you explicitly activate a cloud API.

### New — AI backbone

- **Seven provider adapters, one interface.** `AIProvider` abstraction; three kinds (HTTP local, HTTP remote, CLI spawn). All stream chunks through an `AsyncIterable` so the UI renders tokens as they land.
- **Ollama** — HTTP adapter on `localhost:11434`. NDJSON streaming from `/api/generate`. Zero config; model list fetched live from `/api/tags`.
- **Anthropic API** — SSE streaming via `/v1/messages`. Model list fetched live from `/v1/models` with a safe fallback.
- **OpenAI API** — SSE streaming via `/v1/chat/completions`. Model list fetched from `/v1/models`, filtered to chat-capable families. Base URL overridable via `localStorage["typex:ai:openai-base"]` for OpenRouter / Together / Groq / LiteLLM proxies.
- **Gemini API** — `streamGenerateContent?alt=sse`. Model list fetched from `/v1beta/models` and filtered to entries advertising `generateContent`.
- **Claude Code CLI** — auto-detected via `cli_which`. Spawned as `claude --print --model <id>` with the prompt on stdin; stdout streams line-by-line.
- **Codex CLI** — auto-detected. Spawned as `codex exec --model <id>` with stdin prompt.
- **Gemini CLI** — auto-detected. Spawned as `gemini --model <id> -p -`.
- **OS keychain for API keys** — Windows Credential Manager, macOS Keychain, or Secret Service on Linux, via the Rust `keyring` crate (service name `TypeX`). Keys never touch the preferences file. Legacy localStorage values migrate once on first read and the plaintext copy is wiped.
- **Custom model IDs** — `+ Custom…` button on every model picker. Persisted per-provider in localStorage; use it for pre-release IDs, fine-tunes, or OpenRouter routing strings.

### New — writing surfaces

- **Inline ghost-text autocomplete.** After a pause (default 700 ms, adjustable 250–2500 ms in Preferences), a dim italic continuation appears after the cursor. `Tab` accepts, `Esc` dismisses, any keystroke cancels. A strict completion-engine system prompt (user-editable textarea with a reset button) plus a narrow meta-response scrubber keep chat models from introducing themselves instead of continuing your prose. Off by default.
- **In-place edits with a pulsing glow.** Rewrite, Fix grammar, Translate, and Continue don't open a panel. The selection pulses in accent indigo while the model thinks; the result streams in at a blinking caret, replacing the selection as it arrives. `Esc` aborts and restores the original; `Ctrl+Z` after finish undoes (transactions auto-group into one history step).
- **Right-click menu on selections** — compact floating menu with a model-name header (`claude-haiku-4.5 · claude-code`) and the four in-place commands. Only intercepts when AI is on and something's selected; otherwise the browser's native copy/paste menu runs.
- **Discreet status-bar pill.** Just a dot + "AI". Active model details live in the tooltip, the right-click menu header, and Preferences. Pulses accent during in-flight requests; click to cancel.

### New — palette commands

- **`AI: Rewrite selection`** — streams a rewrite over the selected text for clarity and rhythm. Voice + markdown preserved.
- **`AI: Fix grammar`** — corrects spelling / grammar / punctuation in place. Deliberately doesn't touch style or structure.
- **`AI: Translate selection…`** — prompts for a target language, then streams the translation in place. Preserves all markdown formatting.
- **`AI: Continue from here`** — streams a continuation at the cursor using the last 2 000 chars of preceding context. Auto-prefixes a space if the cursor is mid-word.
- **`AI: Summarize`** — summarizes the selection (or the first 8 k chars of the doc if nothing is selected). Copies the summary to the clipboard with a progress toast; never overwrites your prose.
- **`AI: Toggle inline autocomplete`** — quick on/off with a confirmation toast.

### New — Preferences → AI

- **Vendor-grouped provider list** — Anthropic API and Claude Code CLI sit under one "Anthropic" header; Codex CLI under "OpenAI"; Gemini CLI under "Google"; Ollama standalone. An intro hint makes the API-vs-CLI choice obvious.
- **Status dot + detail per row** — green when detected, detail line shows the version string for CLIs and the model count for APIs.
- **API key input** with `Save key` — stores in the OS keychain on save.
- **Model picker + "Active" pill** — picking a model updates the button state in place; no more whole-section rerender flashing the list away.
- **Autocomplete toggle + delay slider + system-prompt textarea** — all three persisted in prefs. "Reset to default" restores the strict completion prompt.

### Changed

- **No more bottom-docked AI panel.** Rewrite / Fix / Translate / Continue all run inline with the glow-and-caret animation. Summarize goes straight to clipboard. The panel code is still in tree for reference but is no longer wired up.
- **Provider list** now gets live model counts from the vendor's list endpoint with a hardcoded fallback — no more stale IDs blocking you.

### Fixed

- **Preferences rerender on model change** — picking a new model or activating a provider no longer tears down and rebuilds the whole AI section. Only the affected "Active" button text + disabled state updates, so the list doesn't jump or flash.
- **Autocomplete gating** — removed a too-aggressive gate that blocked all CLI providers from ghost text. The meta-response scrubber handles chat-tuned models on its own now; worst case is no ghost text, never a bad one.

### Install

Download **[TypeX-latest-x64.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX-latest-x64.msi)** or the versioned **[TypeX_0.4.0_x64_en-US.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX_0.4.0_x64_en-US.msi)** and run it.

- **Size:** ~45 MB download, ~225 MB on disk (most of it Pandoc 3.9).
- **Requires:** Windows 10 or 11, x64. WebView2 auto-installs on first run if needed.
- **Upgrades cleanly in place** from any 0.1.x, 0.3.x, or 0.4.x.
