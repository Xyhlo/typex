**TypeX 0.1.1 — Hotfix**

Two paste-related fixes on top of the 0.1.0 beta.

### Fixes

- **Pasting source code now produces a single syntax-highlighted code block.** Previously, pasting raw Python / JS / shell / etc. into a markdown document produced a jumbled mix of paragraphs (unindented lines) and 4-space-indented code blocks (CommonMark quirk) — no language, no colors. Now TypeX detects code vs. markdown, auto-detects the language (shebang fast paths plus lowlight/hljs relevance scoring), and wraps the paste in a fenced block before parsing. Works for Python, JavaScript/TypeScript, Bash, Ruby, Rust, Go, Java, C/C++, C#, PHP, HTML, CSS, JSON, YAML, and more.
- **Status bar mode label now reflects the document's source format** (Word, OpenDocument, LaTeX, EPUB, AsciiDoc, reStructuredText, Org mode, etc.) instead of a hardcoded "Markdown" that never updated.
- **Clicking "New document" no longer spawns multiple tabs.** The sidebar buttons were stacking a new click listener on every file-tree re-render via a MutationObserver — one click eventually fired a dozen `createNew()` calls. Buttons now carry a `data-wired` flag so they only bind once.

### Install

Download **[TypeX-latest-x64.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX-latest-x64.msi)** or the versioned **[TypeX_0.1.1_x64_en-US.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX_0.1.1_x64_en-US.msi)** and run it. If you installed 0.1.0, Windows will upgrade in place — no uninstall needed.

Starting this release, the `TypeX-latest-x64.msi` filename is stable across future releases, so direct links and the website's Download button won't break when versions change.
