**TypeX 0.1.2 — Windows integration + animations**

Biggest update since the initial beta. Windows Explorer now shows branded per-extension icons for any file type where TypeX is the default handler, a first-run prompt helps you get there, and in-app progress spinners land on every Pandoc operation.

### New

- **Per-extension file icons in Explorer.** 20 icon designs covering 26 extensions — MD, DOCX, ODT, RTF, HTML, EPUB, RST, ADOC, TEX, ORG, TXTL, WIKI, MUSE, T2T, IPYNB, BIB, FB2, OPML, MAN, TXT. Purple TypeX tile with the extension name overlaid. Icons ship with the installer; they appear in Explorer the moment you pick TypeX as the default for that file type.
- **First-run default-app prompt.** When you launch TypeX for the first time (or after a fresh install), a non-blocking banner slides in beneath the titlebar offering to make TypeX the default for Markdown — or, if you opened the app from a `.docx` / `.epub` / any supported file, it adapts to ask about *that* type specifically. One click opens Windows Settings → Default apps on the right page.
- **Preferences → Windows integration.** New row with an **Open Default Apps settings…** button for managing defaults any time later.
- **Progress toast with spinners.** Pandoc imports, exports, Save As to non-markdown, and revert now show a persistent toast with a ring spinner. On completion it resolves to a green checkmark (or a red error if something went wrong) instead of instantly disappearing.
- **Tab-switch fade.** Brief 160 ms opacity dip while content swaps so tab switches don't pop.
- **About modal** uses a proper inline SVG mark that's sharp at any DPI and rebinds to the active theme's accent.
- **Sidebar auto-collapse** when launched with a file (from Explorer, "Open with", drag-drop on the .exe). The user asked to see one file, so the editor gets full width; Ctrl+\\ brings the sidebar back.
- **Design-system folder** (`design-system/`) — foundations, components, brand marks, 20 loading animations, overview page with theme toggle. Everything `--tx-*`-tokenized.

### Fixed

- **Paste of source code** now produces a single syntax-highlighted code block instead of a mix of paragraphs and 4-space-indented code blocks (earlier fix from 0.1.1, confirmed in 0.1.2).
- **Status-bar mode label** reflects the document's actual source format (Word, OpenDocument, LaTeX, etc.) instead of a hardcoded "Markdown."
- **Sidebar New-document button** no longer stacks click listeners — one click, one tab, always.
- **MSI upgrade from 0.1.0 / 0.1.1** now works in place without an uninstall step.

### Install

Download **[TypeX-latest-x64.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX-latest-x64.msi)** or the versioned **[TypeX_0.1.2_x64_en-US.msi](https://github.com/Xyhlo/typex/releases/latest/download/TypeX_0.1.2_x64_en-US.msi)** and run it.

The `TypeX-latest-x64.msi` filename is stable across releases, so direct links and the website's Download button keep working.

- **Size:** 45 MB download, ~225 MB on disk (most of it Pandoc 3.9).
- **Requires:** Windows 10 or 11, x64. WebView2 auto-installs on first run if needed.
